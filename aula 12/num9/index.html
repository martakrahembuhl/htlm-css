HTML Semântico: Conceitos Básicos

O que significa HTML Semântico?

HTML Semântico é a prática de usar tags HTML para reforçar o significado do conteúdo, em vez de simplesmente definir sua aparência. Tags como <header>, <footer>, <article> e <section> dão contexto ao navegador e a ferramentas de acessibilidade.

Qual a diferença entre <div> e <section>?

A tag <div> é um contêiner genérico, sem significado semântico. Já a tag <section> representa uma seção temática agrupada do documento. A <section> deve ter, idealmente, um cabeçalho (<h1> a <h6>) para definir seu tema.

Para que serve a tag <aside>?

A tag <aside> é usada para conteúdo que está relacionado ao conteúdo principal da página, mas que pode ser considerado separado. Exemplos incluem barras laterais, caixas de citação, anúncios ou informações biográficas do autor.

Quando devo usar <article>?

Use <article> para qualquer conteúdo que seja independente e que poderia ser distribuído ou reutilizado em outro lugar, como uma postagem de blog, um comentário, um artigo de jornal ou um item em um widget.

O uso de tags semânticas afeta o SEO?

Sim. Motores de busca (Google, Bing) usam a estrutura semântica para entender melhor a relevância e o contexto do seu conteúdo. Uma estrutura bem definida ajuda os robôs a indexar sua página de forma mais eficiente.
